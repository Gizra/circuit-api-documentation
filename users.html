Users & Staff

Endpoints for managing staff users, creating accounts, and password operations.

Available Endpoints

MethodEndpointDescription
GET/api/v1.0/staffList staff members
GET/api/v1.0/staff/{id}Get a single staff member
POST/api/v1.0/users-updateCreate / update user account
POST/api/v1.0/reset-passwordSend password reset email
GET /api/v1.0/staff
List Staff

Returns a paginated list of staff users. Supports filtering by role and status.

Query Parameters
ParameterTypeDescription
rangeintegerNumber of records to return per page.
pageintegerPage number (1-based).
filter[roles]stringFilter by role name (e.g. administrator, clerk).
filter[status]stringFilter by account status (e.g. active, blocked).
Example Request
curl -X GET "https://backoffice.ddev.site/api/v1.0/staff?filter[roles]=clerk&range=25&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "5",
      "name": "Jane Smith",
      "email": "jane@example.com",
      "roles": ["administrator", "clerk"]
    }
  ],
  "count": 15
}
GET /api/v1.0/staff/{id}
Get Staff Member

Returns the full record for a single staff member identified by their user ID.

Path Parameters
ParameterTypeDescription
idintegerThe ID of the staff member to retrieve.
Example Request
curl -X GET "https://backoffice.ddev.site/api/v1.0/staff/5" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": {
    "id": "5",
    "name": "Jane Smith",
    "email": "jane@example.com",
    "roles": ["administrator", "clerk"],
    "status": "active",
    "created": "2024-01-15T10:00:00Z"
  }
}
POST /api/v1.0/users-update
Create / Update User

Creates a new user account or updates an existing one. Authentication is optional — anonymous creation is allowed for public-facing registrations (e.g. bidder self-registration). When called without a Bearer token, role assignment is ignored.

Request Body
FieldTypeRequiredDescription
namestringYesUsername (machine name, used for login).
mailstringYesEmail address of the user.
passstringNoPlain-text password. Omit to leave the existing password unchanged.
rolesarray of stringsNoRole machine names to assign to the user. Requires authentication.
field_first_namestringNoUser's first name.
field_last_namestringNoUser's last name.
Example Request — authenticated staff creation
curl -X POST "https://backoffice.ddev.site/api/v1.0/users-update" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "john.doe",
    "mail": "john@example.com",
    "pass": "s3cur3P@ss",
    "roles": ["clerk"],
    "field_first_name": "John",
    "field_last_name": "Doe"
  }'
Example Request — anonymous public registration
curl -X POST "https://backoffice.ddev.site/api/v1.0/users-update" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "jane.bidder",
    "mail": "jane@example.com",
    "pass": "myP@ssword",
    "field_first_name": "Jane",
    "field_last_name": "Bidder"
  }'
Response
{
  "data": {
    "id": "42",
    "name": "john.doe",
    "mail": "john@example.com"
  }
}
POST /api/v1.0/reset-password
Reset Password

Sends a password reset email to the address provided. The email contains a one-time link the user can follow to set a new password.

Rate Limiting — This endpoint is rate limited to protect against abuse:
  • Authenticated users and anonymous callers: 10 requests per day (per user account or originating IP address).
  • Administrators: unlimited.
Requests exceeding the limit will receive a 429 Too Many Requests response.
Request Body
FieldTypeRequiredDescription
mailstringYesThe email address associated with the account that needs a password reset.
Example Request
curl -X POST "https://backoffice.ddev.site/api/v1.0/reset-password" \
  -H "Content-Type: application/json" \
  -d '{
    "mail": "john@example.com"
  }'
Response
{
  "data": {
    "message": "Password reset instructions have been sent."
  }
}
Error Response — rate limit exceeded
{
  "error": {
    "status": 429,
    "message": "Too many requests. Please try again later."
  }
}

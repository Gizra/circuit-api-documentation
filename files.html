Files

Endpoints for retrieving and uploading private files.

GET /api/v1.0/get-file/{id}

Returns the URL and metadata for a private file. The id path parameter is the file entity ID. The actual file content is served via a separate authenticated download URL included in the response.

Parameters
ParameterTypeDescription
idstring (path)File entity ID
Example Request
GET /api/v1.0/get-file/88
Response
{
  "data": {
    "id": "88",
    "filename": "invoice_456.pdf",
    "url": "/system/files/invoice_456.pdf",
    "mime": "application/pdf",
    "size": 204800,
    "created": "2025-10-01T10:00:00Z"
  }
}
POST /api/v1.0/private-files

Uploads a file to the private file system. The request must be sent as multipart/form-data. The uploaded file will not be publicly accessible.

File upload must be enabled on the server via the restful_file_upload server variable. Contact your system administrator if uploads return a 403 or 501 response.
Request Headers
HeaderValue
Content-Typemultipart/form-data
Request Body (multipart/form-data)
ParameterTypeDescription
filebinary (required)The file to upload
filenamestring (optional)Override the stored filename; defaults to the uploaded file's name
Example Request
POST /api/v1.0/private-files
Content-Type: multipart/form-data

curl -X POST https://backoffice.ddev.site/api/v1.0/private-files \
  -H "Authorization: Bearer {token}" \
  -F "file=@/path/to/document.pdf" \
  -F "filename=document.pdf"
Response
{
  "data": {
    "id": "89",
    "filename": "document.pdf",
    "url": "/system/files/document.pdf",
    "size": 102400
  }
}

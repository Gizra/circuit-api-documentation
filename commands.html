Commands

Action-oriented endpoints for invoking server functions, bulk operations, and print/export utilities.

These endpoints trigger server-side operations. Most are not discoverable via the API index. All require authentication unless noted.
POST /api/v1.0/invoke-function

Invokes a named server-side function. The available functions depend on server configuration. The response format is a raw passthrough of the function's return value and will vary by function.

Request Body
ParameterTypeDescription
functionstring (required)Machine name of the function to invoke
argsobject (optional)Key-value arguments passed to the function
Example Request
POST /api/v1.0/invoke-function
{
  "function": "recalculate_sale_totals",
  "args": {
    "sale_id": "45"
  }
}
Response
{
  "data": {
    "status": "ok",
    "message": "Sale totals recalculated for sale 45."
  }
}
POST /api/v1.0/invoke-function-public

Same as invoke-function but accessible without a Bearer token. Permission checks are performed inside the function itself — the function is responsible for validating whether the caller is authorized to execute it.

Request Body
ParameterTypeDescription
functionstring (required)Machine name of the function to invoke
argsobject (optional)Key-value arguments passed to the function
Example Request
POST /api/v1.0/invoke-function-public
{
  "function": "get_public_sale_summary",
  "args": {
    "sale_id": "12"
  }
}
Response
{
  "data": {
    "sale_id": "12",
    "total_lots": 385,
    "status": "upcoming"
  }
}
POST /api/v1.0/bulk-dispatcher

Generic endpoint for dispatching bulk operations across multiple entities. See the dedicated Bulk Dispatcher page for a full list of supported operations and their parameters.

Request Body
ParameterTypeDescription
operationstring (required)Machine name of the bulk operation to run
idsarray (required)Array of entity node IDs to operate on
paramsobject (optional)Additional operation-specific parameters
Example Request
POST /api/v1.0/bulk-dispatcher
{
  "operation": "mark_items_sold",
  "ids": ["100", "101", "102"],
  "params": {
    "sale_id": "45"
  }
}
Response
{
  "data": {
    "processed": 3,
    "errors": []
  }
}
POST /api/v1.0/generate-cheque

Generates a cheque document for a given transaction. Returns a URL to the generated PDF.

Request Body
ParameterTypeDescription
transaction_idstring (required)Node ID of the transaction for which to generate the cheque
Example Request
POST /api/v1.0/generate-cheque
{
  "transaction_id": "456"
}
Response
{
  "data": {
    "cheque_url": "/files/cheque_456.pdf"
  }
}
PATCH /api/v1.0/reset-statement-charges/{id}

Removes all current charges from the specified consignment statement and re-assigns all unassigned charges to it. The id path parameter is the numeric consignment statement node ID.

Parameters
ParameterTypeDescription
idinteger (path)Consignment statement node ID
Example Request
PATCH /api/v1.0/reset-statement-charges/789
Response
{
  "data": {
    "id": "789",
    "status": "charges_reset"
  }
}

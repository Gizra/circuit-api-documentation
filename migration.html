Migration

Endpoints for inspecting migration handler definitions, groups, and reviewing the result logs produced by item, bid, and message migrations.

GET /api/v1.0/migrates

Lists all available migration handler definitions registered in the system. Each handler describes a discrete migration process and its current execution state.

Example Request
curl -X GET https://your-domain.com/api/v1.0/migrates \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "1",
      "machine_name": "client_migration",
      "label": "Client Migration",
      "status": "idle"
    }
  ],
  "count": 8
}
GET /api/v1.0/migrate-groups

Returns all migration groups. Groups are used to organise related migration handlers and control execution order.

Example Request
curl -X GET https://your-domain.com/api/v1.0/migrate-groups \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "1",
      "name": "default",
      "description": "Default migration group"
    }
  ]
}
GET /api/v1.0/migrate-items

Returns paginated result logs for item migrations. Each record represents a single source item and its import outcome.

Query Parameters
ParameterTypeRequiredDescription
rangenumberNoNumber of results per page.
pagenumberNoPage number (1-based).
filter[status]stringNoFilter by migration status (e.g. imported, failed, skipped).
Example Request
curl -X GET "https://your-domain.com/api/v1.0/migrate-items?filter[status]=imported&range=50&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "1",
      "source_id": "OLD-123",
      "status": "imported",
      "created": "2025-01-10T10:00:00Z"
    }
  ],
  "count": 1240
}

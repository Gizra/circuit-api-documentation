GET /api/v1.0/bids_confirmation
List Bid Confirmations

Returns a paginated list of bid confirmations. Use filters to narrow results by sale.

Query Parameters
ParameterTypeDescription
rangeintegerNumber of records to return per page.
pageintegerPage number (1-based).
filter[sale_id]integerFilter confirmations belonging to a specific sale.
Example Request
curl -X GET "https://backoffice.ddev.site/api/v1.0/bids_confirmation?filter[sale_id]=45&range=25&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "1",
      "client_id": "123",
      "sale_id": "45",
      "status": "confirmed",
      "created": "2025-10-01T12:00:00Z"
    }
  ],
  "count": 50
}
POST /api/v1.0/bids_confirmation
Create Bid Confirmation

Creates a new bid confirmation record for a client, sale, and set of lots. A confirmation message or email is typically triggered downstream.

Request Body
FieldTypeRequiredDescription
client_idintegerYesID of the client receiving the confirmation.
sale_idintegerYesID of the sale the confirmation relates to.
lot_idsarray of integersYesArray of lot IDs to include in the confirmation.
Example Request
curl -X POST "https://backoffice.ddev.site/api/v1.0/bids_confirmation" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": 123,
    "sale_id": 45,
    "lot_ids": [101, 102, 105]
  }'
Response
{
  "data": {
    "id": "2",
    "client_id": "123",
    "sale_id": "45",
    "lot_ids": [101, 102, 105],
    "status": "confirmed",
    "created": "2025-10-02T09:30:00Z"
  }
}
GET /api/v1.0/consignments-underbids-inquiries
List Consignment Underbid Inquiries

Returns a paginated list of underbid inquiries associated with consignments. Underbid inquiries are created when a consignor requests information about lots that did not meet the reserve price.

Query Parameters
ParameterTypeDescription
rangeintegerNumber of records to return per page.
pageintegerPage number (1-based).
filter[consignment_id]integerFilter inquiries by consignment ID.
Example Request
curl -X GET "https://backoffice.ddev.site/api/v1.0/consignments-underbids-inquiries?filter[consignment_id]=88&range=25&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "1",
      "consignment_id": "88",
      "item_id": "200",
      "status": "pending"
    }
  ],
  "count": 12
}
PATCH /api/v1.0/consignments-underbids-inquiries/{id}
Generate Underbid Report

Triggers report generation for the specified underbid inquiry. Once processed, the response includes a URL to the generated PDF report.

Path Parameters
ParameterTypeDescription
idintegerThe ID of the underbid inquiry for which to generate the report.
Example Request
curl -X PATCH "https://backoffice.ddev.site/api/v1.0/consignments-underbids-inquiries/1" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json"
Response
{
  "data": {
    "id": "1",
    "report_url": "/files/underbid_report_1.pdf"
  }
}
GET /api/v1.0/general-email-notifications
List General Email Notifications

Returns a paginated list of general email notifications sent by the system. Supports filtering by delivery status and recipient address.

Query Parameters
ParameterTypeDescription
rangeintegerNumber of records to return per page.
pageintegerPage number (1-based).
filter[status]stringFilter by delivery status (e.g. sent, failed, pending).
filter[recipient_email]stringFilter notifications sent to a specific email address.
Example Request
curl -X GET "https://backoffice.ddev.site/api/v1.0/general-email-notifications?filter[status]=sent&range=25&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "5",
      "subject": "Your invoice is ready",
      "recipient": "client@example.com",
      "status": "sent",
      "sent_at": "2025-10-10T08:00:00Z"
    }
  ],
  "count": 320
}
GET /api/v1.0/phone_bidder_cards
List Phone Bidder Cards

Returns a paginated list of phone bidder cards. Cards record the phone number and paddle assigned to a client who will bid by phone during a given sale session.

Query Parameters
ParameterTypeDescription
rangeintegerNumber of records to return per page.
pageintegerPage number (1-based).
filter[sale_id]integerFilter cards by sale ID.
filter[session_id]integerFilter cards by session ID.
Example Request
curl -X GET "https://backoffice.ddev.site/api/v1.0/phone_bidder_cards?filter[sale_id]=45&range=25&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "10",
      "sale_id": "45",
      "client_id": "123",
      "paddle_number": "42",
      "phone": "+1-555-0100"
    }
  ],
  "count": 80
}
POST /api/v1.0/phone_bidder_cards
Create / Update Phone Bidder Card

Creates a new phone bidder card or updates an existing one. If a card already exists for the given client and session, it will be updated in place.

Request Body
FieldTypeRequiredDescription
sale_idintegerYesID of the sale.
client_idintegerYesID of the client who will bid by phone.
session_idintegerYesID of the sale session.
paddle_numberstringYesPaddle number assigned to the phone bidder.
phonestringYesPhone number to call during the session.
lotsarray of integersNoArray of lot IDs the client intends to bid on.
Example Request
curl -X POST "https://backoffice.ddev.site/api/v1.0/phone_bidder_cards" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "sale_id": 45,
    "client_id": 123,
    "session_id": 7,
    "paddle_number": "42",
    "phone": "+1-555-0100",
    "lots": [201, 205, 210]
  }'
Response
{
  "data": {
    "id": "10",
    "sale_id": "45",
    "client_id": "123",
    "session_id": "7",
    "paddle_number": "42",
    "phone": "+1-555-0100",
    "lots": [201, 205, 210]
  }
}
DELETE /api/v1.0/phone_bidder_cards
Delete Phone Bidder Card

Deletes one or more phone bidder cards. Supply a single id or an array of ids to delete in bulk.

Request Body
FieldTypeRequiredDescription
idintegerOne of id or idsID of a single card to delete.
idsarray of integersOne of id or idsArray of card IDs to delete in bulk.
Example Request — single
curl -X DELETE "https://backoffice.ddev.site/api/v1.0/phone_bidder_cards" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{ "id": 10 }'
Example Request — bulk
curl -X DELETE "https://backoffice.ddev.site/api/v1.0/phone_bidder_cards" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{ "ids": [10, 11, 12] }'
Response
{
  "data": {
    "deleted": true
  }
}

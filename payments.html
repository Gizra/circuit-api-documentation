Payments

Endpoints for payment processing, payment methods, cheque generation, and statement charge management.

POST /api/v1.0/payment

Process a payment for a client. Optionally link the payment to one or more existing orders.

Body Parameters
ParameterTypeRequiredDescription
client_idstringYesThe ID of the client making the payment.
amountnumberYesPayment amount (decimal).
payment_method_idstringYesID of the payment method to charge.
order_idsarrayNoList of order IDs to apply this payment to.
notesstringNoOptional notes attached to the payment record.
Example Request
curl -X POST https://your-domain.com/api/v1.0/payment \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "123",
    "amount": 500.00,
    "payment_method_id": "pm_001",
    "order_ids": ["order_1", "order_2"],
    "notes": "Payment for October invoices"
  }'
Response
{
  "data": {
    "id": "456",
    "amount": 500.00,
    "status": "completed",
    "transaction_id": "txn_abc123"
  }
}

GET /api/v1.0/payment/transaction/{id}

Retrieve the details of a specific payment transaction by its ID.

Path Parameters
ParameterTypeRequiredDescription

Example Request
curl -X GET https://your-domain.com/api/v1.0/payment/transaction/456 \
  -H "Authorization: Bearer {token}"

  }
}
POST /api/v1.0/payment/refund/{id}

Issue a refund for a completed transaction. An optional reason can be provided for record-keeping.

Path Parameters
ParameterTypeRequiredDescription
idstringYesThe transaction ID to refund.
Body Parameters
ParameterTypeRequiredDescription

Example Request
curl -X POST https://your-domain.com/api/v1.0/payment/refund/456 \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "Item returned by buyer"
  }'
Response
{
  "data": {
    "id": "456",
    "status": "refunded",
    "refunded_at": "2025-10-16T09:00:00Z"
  }
}
GET /api/v1.0/payment-methods

Returns a paginated list of payment methods. Filter by client to retrieve methods associated with a specific client.

Query Parameters
ParameterTypeRequiredDescription
rangenumberNoNumber of results per page.
pagenumberNoPage number (1-based).
filter[client_id]stringNoFilter payment methods by client ID.
Example Request
curl -X GET "https://your-domain.com/api/v1.0/payment-methods?filter[client_id]=123&range=20&page=1" \
  -H "Authorization: Bearer {token}"
Response
{
  "data": [
    {
      "id": "1",
      "label": "Credit Card",
      "type": "cc"
    }
  ],
  "count": 5
}
POST /api/v1.0/payment-methods

Create a new payment method for a client.

Body Parameters
ParameterTypeRequiredDescription
client_idstringYesThe ID of the client this method belongs to.
typestringYesPayment method type (e.g. cc, bank, cash).
labelstringYesHuman-readable label for this payment method.
detailsobjectYesType-specific details (e.g. card last four digits, bank account info).
Example Request
curl -X POST https://your-domain.com/api/v1.0/payment-methods \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "123",
    "type": "cc",
    "label": "Visa ending 4242",
    "details": {
      "last_four": "4242",
      "expiry": "12/27"
    }
  }'
Response
{
  "data": {
    "id": "42",
    "client_id": "123",
    "type": "cc",
    "label": "Visa ending 4242"
  }
}

PATCH /api/v1.0/reset-statement-charges/{id}

Removes current charges from the given consignment statement and re-assigns all unassigned charges to it. Use this endpoint to reconcile a statement after charges have been modified or reassigned elsewhere.

Path Parameters
ParameterTypeRequiredDescription
idstringYesThe consignment statement node ID.
Example Request
curl -X PATCH https://your-domain.com/api/v1.0/reset-statement-charges/789 \
  -H "Authorization: Bearer {token}"
Response
{
  "data": {
    "id": "789",
    "status": "charges_reset"
  }
}
